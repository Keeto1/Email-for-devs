Educational Concept

This documentation represents a conceptual API design created for educational purposes. The examples demonstrate modern API patterns and developer experience principles.

Quick Start

Get up and running with FAFCODE in minutes. This example shows how the integration would work.

1. Install the SDK

# npm
npm install @fafcode/sdk

# yarn  
yarn add @fafcode/sdk

# pnpm
pnpm add @fafcode/sdk

2. Initialize Client

import { FafcodeClient } from '@fafcode/sdk';

const client = new FafcodeClient({
  apiKey: 'your-api-key',
  environment: 'production'
});

3. Send Your First Email

const result = await client.emails.send({
  to: 'user@example.com',
  subject: 'Welcome to our platform!',
  template: 'welcome',
  variables: {
    firstName: 'John',
    loginUrl: 'https://app.example.com/login'
  }
});

console.log('Email sent:', result.id);

Authentication

FAFCODE would use API keys for authentication. Here's how it would work:

API Key Authentication

Include your API key in the request headers:

curl -X POST https://api.fafcode.com/v1/emails/send \
  -H "Authorization: Bearer your-api-key" \
  -H "Content-Type: application/json" \
  -d '{
    "to": "user@example.com",
    "template": "welcome"
  }'

Production Keys

Start with faf_live_

Used for sending real emails

Test Keys

Start with faf_test_

Safe for development and testing

Send Emails API

The core API for sending transactional emails.

POST /v1/emails/send

Send a transactional email using a template or raw HTML content.

Request Body

{
  "to": "string | string[]",
  "from": "string (optional)",
  "subject": "string (optional if template used)",
  "template": "string (optional)",
  "variables": "object (optional)",
  "html": "string (optional)",
  "text": "string (optional)",
  "attachments": "array (optional)",
  "tags": "string[] (optional)",
  "metadata": "object (optional)"
}

Response

{
  "id": "email_1234567890",
  "status": "sent",
  "to": ["user@example.com"],
  "from": "noreply@yourapp.com",
  "subject": "Welcome to our platform!",
  "template": "welcome",
  "created_at": "2025-01-09T10:30:00Z",
  "delivery_time": "0.342s"
}

Error Responses

400 Bad Request
{ "error": "Invalid email format", "code": "INVALID_EMAIL" }
401 Unauthorized
{ "error": "Invalid API key", "code": "UNAUTHORIZED" }

SDK Examples

Code examples for different programming languages.

JS Node.js 

// npm install @fafcode/node
import { FafcodeClient } from '@fafcode/node';

const client = new FafcodeClient('your-api-key');

// Send email with template
const result = await client.emails.send({
  to: 'user@example.com',
  template: 'welcome',
  variables: { name: 'John' }
});

// Send raw HTML email
const result2 = await client.emails.send({
  to: 'user@example.com',
  subject: 'Custom Email',
  html: '<h1>Hello World</h1>'
});

Best Practices

Guidelines for effective email development and delivery.

Error Handling

  • • Always implement retry logic
  • • Handle rate limiting gracefully
  • • Log errors for debugging
  • • Provide user-friendly error messages

Performance

  • • Use templates for better performance
  • • Batch multiple recipients when possible
  • • Implement caching for templates
  • • Monitor delivery metrics

Security

  • • Keep API keys secure
  • • Use environment variables
  • • Validate input data
  • • Implement rate limiting

Deliverability

  • • Use proper sender authentication
  • • Maintain clean recipient lists
  • • Follow email best practices
  • • Monitor bounce rates

Testing

How to test your email integration effectively.

Test Mode

Use test API keys to avoid sending real emails during development:

const client = new FafcodeClient({
  apiKey: 'faf_test_1234567890',  // Test key
  environment: 'test'
});

// This won't send real emails
const result = await client.emails.send({
  to: 'test@example.com',
  template: 'welcome'
});

Unit Testing

Example unit test using Jest:

// email.test.js
import { EmailService } from './email-service';

describe('EmailService', () => {
  test('should send welcome email successfully', async () => {
    const emailService = new EmailService('test-key');
    const user = { email: 'test@example.com', name: 'John' };
    
    const result = await emailService.sendWelcomeEmail(user);
    
    expect(result.status).toBe('sent');
    expect(result.to).toContain(user.email);
  });
});
